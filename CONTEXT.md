# 领域语言

## 核心实体

### 智能旅行箱（Box）
平台持有的物理设备，内置多种传感器（温感、GPS、摄像头、麦克风、加速度计、光感）。箱子是平台的核心资产。

**状态：** 可用 → 借出 → 检查中 → 可用

- **可用**：箱子在驿站待取，未与任何行程绑定
- **借出**：箱子已与宠物主绑定，处于行程中
- **检查中**：宠物主已还箱，驿站检查箱子外观和设备状态中
- **离线**：设备断线，需运维介入

### 行程（Trip）
一次宠物托运过程的记录。一个行程关联一个箱子、一个宠物主、和一段传感器/视频数据。

- 行程开始：驿站扫箱码 + 扫用户码，确认绑定生效
- 行程结束：驿站扫箱码，确认还箱，绑定解除

### 宠物主（Owner）
通过小程序租用箱子的用户。宠物主在行程期间享有箱子的实时监测权限，行程结束后可查看旅程报告。

### 驿站（Station）
宠物店合作点，作为箱子的物理流转节点。驿站操作员负责：取箱确认、还箱确认、箱子外观检查、异常上报。

在互联网镖局模型中，驿站同时是箱子调度节点——箱子在目的地城市驿站等待回流调度。

### 镖师（Peteer）
宠物的护送服务提供者，通过邀请码入驻平台。

镖师是平台的主动提供者，不是被招募的员工：
- **入驻门槛**：押金（硬筛选）+ 身份验证
- **信任积累**：护镖记录代替品牌背书，历史记录是主要信任证明
- **接单模式**：宠物主委托镖师护送特定路段，镖师自主接单
- **博主角色**：博主发邀请码招募镖师，博主用自己的网络声誉为镖师背书；博主不抽成，为爱发电赚名声

镖师分段护送，行程可包含公路段和铁路段。

## 互联网镖局模型

平台构建平权网络，替代传统宠物托运公司的层级结构。

### 核心差异化
- **定价透明**：镖师直接服务宠物主，无中间商加价
- **信任来源**：护镖记录 + 三层数据，而非公司品牌
- **网络结构**：博主发邀请码招募镖师，结构平等，无特权层级

### 镖师网络入驻流程
1. 博主发邀请码给潜在镖师
2. 镖师提交押金 + 身份验证入驻
3. 镖师接单护送宠物，累积护镖记录
4. 博主网络声誉与镖师行为挂钩（出问题时博主可能向粉丝公开道歉）

### 护送段模型
宠物行程由宠物主规划，可分为多段护送，每段委托不同镖师：
- **取送上门段**：镖师A上门取宠物，送至出发地火车站
- **铁路托运段**：宠物+箱子走CRE铁路货运，镖师A不上车
- **接站送段**：镖师B在目的地站接箱子+宠物，送至宠物主

### 三层信任数据
- **轨迹信任**：GPS证明路线正确，无绕路或甩单
- **状态信任**：温湿度/气体浓度证明环境舒适
- **行为信任**：视频证明无粗暴对待

箱子上火车后切快照模式（非视频），节省存储和电量。

### 箱子调度
箱子全程跟宠物走。铁路托运时箱子与宠物同车同步到达目的地站。

目的地驿站是箱子调度节点：
- 箱子优先等待自然循环回流（目的地→出发地反向需求）
- 邻近节点调拨作为补充（平台承担运费）
- 早期在一线和省会城市试点

## 业务流程

### 取箱
1. 宠物主在小程序预约箱子，选择取箱驿站
2. 宠物主到驿站，驿站操作员扫箱子二维码 + 扫宠物主小程序码
3. 绑定生效，行程开始，宠物主可实时监测

### 还箱
1. 宠物主到驿站还箱
2. 驿站操作员扫箱子码，确认还箱
3. 绑定解除，箱子进入"检查中"
4. 驿站检查箱子外观和设备状态
5. 状态正常则标记为"可用"

## 监测数据

### 传感器数据类型
- **温感**：环境温度，单位 °C
- **GPS**：经纬度坐标
- **加速度计**：冲击事件，单位 g
- **麦克风**：音频摘要（异常声音标记）
- **光感**：开盖事件（0/1）

### 视频片段
15 分钟本地录制，分段上传，云端保留 7 天。

**录制模式切换**：箱子上火车后自动切快照模式（每 N 分钟一张），节省存储和电量。

### 快照
周期性图片，独立于视频上传，用于快速确认宠物状态。

## 告警类型

| 告警 | 描述 |
|------|------|
| 温度告警 | 环境温度超出阈值 |
| 声音异常告警 | 检测到持续异常声音（吠叫/哀鸣） |
| 冲击告警 | 加速度瞬时超过冲击阈值 |
| 开盖告警 | 检测到非授权开盖 |
| 信号中断告警 | 设备离线超过 N 分钟 |

## 旅程报告

行程结束后生成的完整记录，包含：
- GPS 轨迹回放
- 温感曲线（标注异常区间）
- 快照时间线
- 异常事件时间线
- 视频片段可下载链接

## 角色权限

| 角色 | 能做什么 |
|------|---------|
| 宠物主 | 预约箱子、查看实时监测、查看告警记录、查看旅程报告 |
| 驿站运营 | 取箱确认、还箱确认、箱子状态检查、异常上报 |
| 平台管理员 | 设备管理、用户管理、告警规则配置、数据统计 |

## 术语对照

| 术语 | 说明 |
|------|------|
| 镖师 | 宠物护送服务提供者（平台C端/B端角色） |
| 驿站 | 宠物店合作点，箱子物理流转节点 |
| 行程粒度绑定 | 箱子与行程绑定，而非与用户永久绑定 |
| 旅程报告 | 行程结束后生成的完整记录文档 |
| 互联网镖局 | 平台的护送服务网络模型，平权结构替代传统托运公司层级 |
| 护镖记录 | 镖师历史护送记录的摘要，代替品牌作为信任证明 |
| 三层信任数据 | 轨迹+状态+行为三层数据，用于建立宠物主对镖师的信任 |

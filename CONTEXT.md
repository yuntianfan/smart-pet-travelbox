# 领域语言

## 核心实体

### 智能旅行箱（Box）
平台持有的物理设备，内置多种传感器（温感、GPS、摄像头、麦克风、加速度计、光感）。箱子是平台的核心资产。

**状态：** 可用 → 借出 → 检查中 → 可用

- **可用**：箱子在分舵待取，未与任何租赁绑定
- **借出**：箱子已与宠物主绑定，处于租赁中
- **检查中**：宠物主已还箱，分舵检查箱子外观和设备状态中
- **离线**：设备断线，需运维介入

### 租赁（Lease）
箱子从分舵借出到归还的完整周期。租赁是外层容器，行程必须在租赁期间内，但租赁可以没有行程（宠物主取箱后改变主意取消）。

- **租赁开始**：宠物主在分舵选好箱子，扫箱子二维码自助绑定 + 押金确认，分舵放行
- **租赁结束**：宠物主到分舵还箱，分舵扫箱码确认还箱，绑定解除

### 行程（Trip）
一次宠物护送过程的记录。一个行程关联一个箱子、一个宠物主、至少一名镖师和一段传感器/视频数据。

- **行程开始**：镖师从出发地（宠物主小区/家）出发
- **行程结束**：镖师护送宠物到达目的地宠物主居住处，宠物主在小程序上确认到达

行程由宠物主规划，分为多段（Segment），每段关联一个或零个镖师护送，段之间自动切换。

行程一定在租赁期间内。一个租赁期间可包含一段或多段行程（常见为一段）。

### 段（Segment）
行程的子划分，由宠物主规划。段有以下类型：
- **镖师段**：有镖师护送的段，镖师在小程序上确认到达，系统检查镖师和箱子定位及照片后完成段切换
- **无镖师段**：如铁路运输段（通过中铁快运预约爱宠单独行），镖师不上车，段自动开始

传感器数据归属段，归属于行程。

### 宠物主（Owner）
通过小程序租用箱子的用户。宠物主在租赁期间享有箱子的实时监测权限，行程结束后可查看旅程报告。

### 分舵（Station）
宠物店合作点，作为箱子的物理流转节点，由舵主（宠物店店长）运营，可安排店员负责具体操作。分舵操作员负责：取箱确认、还箱确认、箱子外观检查、异常上报。

在互联网镖局模型中，分舵同时是箱子调度节点——箱子在目的地城市分舵等待回流调度。

### 镖师（Peteer）
宠物的护送服务提供者，通过分舵舵主邀请码入驻平台。

镖师是平台的主动提供者，不是平台员工：
- **来源**：本地分舵舵主招募的爱宠人士（从熟悉的宠物主中邀请）
- **服务范围**：以短途为主，同城或本市县到邻近市县；镖师可自主选择是否接长途（自驾）
- **身份**：自由职业，镖师自主接单，平台统一调度
- **信任积累**：护镖记录代替品牌背书，历史记录是主要信任证明
- **舵主关系**：舵主招募镖师为分舵提升营收和订单密度，镖师行为由平台管理，舵主不承担连带责任

镖师分段护送，行程可包含公路段和铁路段。

## 互联网镖局模型

平台构建平权网络，替代传统宠物托运公司的层级结构。

### 核心差异化
- **定价透明**：镖师直接服务宠物主，无中间商加价
- **信任来源**：护镖记录 + 三层数据，而非公司品牌
- **网络结构**：舵主发邀请码招募镖师，结构平等，无特权层级

### 镖师网络入驻流程
1. 舵主发邀请码给潜在镖师
2. 镖师提交押金 + 身份验证入驻
3. 镖师接单护送宠物，累积护镖记录
4. 镖师行为由平台管理，舵主为分舵营收发展镖师网络，不承担连带责任

### 行程段模型
宠物行程由宠物主规划，分为多段，每段关联一个或零个镖师护送：
- **取送上门段**：镖师A上门接宠物，送至出发地火车站，镖师确认到达后切换下一段
- **铁路托运段**：宠物+箱子走CRE铁路货运，镖师A不上车，段自动开始
- **接站送段**：镖师B在目的地站接箱子+宠物，送至宠物主居住处

### 三层信任数据
- **轨迹信任**：GPS证明路线正确，无绕路或甩单
- **状态信任**：温湿度/气体浓度证明环境舒适
- **行为信任**：视频证明无粗暴对待

箱子上火车后切快照模式（非视频），节省存储和电量。

### 箱子调度
箱子全程跟宠物走。铁路托运时箱子与宠物同车同步到达目的地站。

目的地分舵是箱子调度节点：
- 箱子优先等待自然循环回流（目的地→出发地反向需求）
- 邻近节点调拨作为补充（平台承担调度费）
- 早期在一线和省会城市试点

### 分舵收益模型
分舵（舵主/宠物店店长）作为分布式仓库节点，其收益来源：
- **租赁抽成**：按平台统一比例从每笔租赁费中分成，用于覆盖库存费和空间成本
- **自营镖师服务**：舵主或店员兼任镖师，接本分舵辐射范围的单，服务费归己
- **镖师手续费抽成**：镖师接单缴纳平台手续费，从中抽成给分舵，激励分舵发展镖师网络

镖师行为由平台管理，舵主不承担连带责任。

## 业务流程

### 取箱
1. 宠物主在小程序预约箱子，选择取箱分舵
2. 宠物主到分舵，选好箱子后扫箱子二维码自助绑定
3. 押金确认（如未交）后，分舵放行，租赁开始

### 还箱
1. 宠物主到分舵还箱
2. 分舵操作员扫箱子码，确认还箱
3. 绑定解除，箱子进入"检查中"
4. 分舵检查箱子外观和设备状态
5. 状态正常则标记为"可用"

## 监测数据

### 传感器数据类型
- **温感**：环境温度，单位 °C
- **GPS**：经纬度坐标
- **加速度计**：冲击事件，单位 g
- **麦克风**：音频摘要（异常声音标记）
- **光感**：开盖事件（0/1）

### 视频片段
15 分钟本地录制，分段上传，云端保留 7 天。

**录制模式切换**：箱子上火车后自动切快照模式（每 N 分钟一张），节省存储和电量。

### 快照
周期性图片，独立于视频上传，用于快速确认宠物状态。

## 告警类型

| 告警 | 描述 |
|------|------|
| 温度告警 | 环境温度超出阈值 |
| 声音异常告警 | 检测到持续异常声音（吠叫/哀鸣） |
| 冲击告警 | 加速度瞬时超过冲击阈值 |
| 开盖告警 | 检测到非授权开盖 |
| 信号中断告警 | 设备离线超过 N 分钟 |

## 旅程报告

行程结束后生成的完整记录，包含：
- GPS 轨迹回放
- 温感曲线（标注异常区间）
- 快照时间线
- 异常事件时间线
- 视频片段可下载链接

## 角色权限

| 角色 | 能做什么 |
|------|---------|
| 宠物主 | 预约箱子、查看实时监测、查看告警记录、查看旅程报告 |
| 舵主 | 招募镖师和宠物主、发展分舵网络、自营镖师服务、查看分舵运营数据 |
| 镖师 | 接单护送宠物、查看护镖记录和收入 |
| 分舵运营 | 取箱确认、还箱确认、箱子状态检查、异常上报 |
| 平台管理员 | 设备管理、用户管理、告警规则配置、数据统计 |

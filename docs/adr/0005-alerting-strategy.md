# ADR-0005: 告警触达策略

**状态:** 已确定  
**日期:** 2026-06-01

## 背景

温度、声音、冲击、开盖等异常事件需要及时触达宠物主。宠物主是低频用户，不会长时间打开小程序，需要多层次触达机制。

## 决策

**多层次触达：微信小程序推送优先 → 3小时未查看则发送短信**

告警类型：
- **温度异常**：高于或低于设定阈值
- **声音异常**：检测到持续吠叫、哀鸣等异常声音
- **冲击事件**：加速度瞬时超过冲击阈值（如粗暴搬运）
- **开盖事件**：光感检测到非授权开盖
- **信号中断**：设备离线超过 N 分钟

## 流程

1. 设备端检测到异常事件，立即上报后台
2. 后台触发告警，发送微信小程序订阅消息/模板消息至宠物主
3. 宠物主查看告警并确认 → 告警标记为"已处理"
4. 宠物主 3 小时内未查看 → 后台触发短信告警
5. 告警记录存入数据库，可在告警历史中查看

## 告警阈值（全局可配置）

| 告警类型 | 阈值 |
|---------|------|
| 温度高 | > 35°C（暂定） |
| 温度低 | < 10°C（暂定） |
| 冲击加速度 | > 3g（瞬时，暂定） |
| 声音异常持续 | > 30秒连续异常（暂定） |

阈值通过后台管理界面可调，不需更新设备固件。

## 理由

- **微信推送优先**：最低成本触达，已在小程序内
- **短信兜底**：3小时未查看可能是用户关闭了通知权限，短信确保最终触达
- **短信限时**：避免过度骚扰用户，也控制短信成本

## 影响

- 后台需要维护用户联系方式（微信 openid + 手机号）
- 短信通道需要接入服务商（如阿里云/腾讯云短信）
- 告警规则在后台可配置，支持按告警类型分别设置阈值

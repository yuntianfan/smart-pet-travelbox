# ADR-0003: 监测传感器栈

**状态:** 已确定  
**日期:** 2026-06-01

## 背景

宠物主需要感知宠物在途状态。不同传感器证明不同维度的"宠物状态 OK"。

## 决策

**MVP 传感器组合：温感 + GPS + 摄像头 + 麦克风 + 加速度计 + 光感。**

| 传感器 | 证明内容 | 上报方式 |
|--------|---------|---------|
| 温感 | 环境温度不过冷/过热 | 5分钟周期上报 |
| GPS | 位置和轨迹 | 5分钟周期上报 |
| 摄像头 | 宠物视觉状态（快照） | 事件触发（周期性快照） |
| 麦克风 | 异常声音（吠叫/哀鸣） | 5分钟周期上报（音频摘要） |
| 加速度计 | 冲击/粗暴搬运 | 5分钟周期上报 + 瞬时事件上报 |
| 光感 | 非授权开盖 | 事件触发上报 |

## 理由

- **温感 + GPS**：基础感知，宠物主最核心的信息需求
- **摄像头快照**：比纯数字更让主人放心，提供视觉确认
- **加速度计**：镖师粗暴搬运是信任痛点，冲击事件是其他传感器无法覆盖的独特证明
- **麦克风**：异常声音（持续吠叫、哀鸣）能反映宠物应激状态
- **光感**：检测非授权开盖，提供物理安全感知

## 未采纳传感器

| 传感器 | 未采纳原因 |
|--------|-----------|
| 称重传感器 | 实现难度中等，MVP 阶段优先级低 |
| CO2/空气质量 | 实现难度高，MVP 阶段不考量 |

## 影响

- 设备端固件需要支持 6 种传感器的数据采集和上报协议
- 后台需要存储和处理 6 种传感器的时序数据
- 设备通信模组需支持 4G Cat.1（已确定），需足够带宽承载音频摘要

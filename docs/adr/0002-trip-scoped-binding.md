# ADR-0002: 设备绑定 — 行程粒度绑定

**状态:** 已确定  
**日期:** 2026-06-01

## 背景

智能旅行箱作为平台核心资产，需要明确与宠物主的关系。PRD 原有描述为永久绑定（零售模式），租赁模式下需要重新定义。

## 决策

**箱子与行程绑定，而非与用户永久绑定。绑定在分舵扫码确认取箱时生效，在分舵扫码确认还箱时解除。**

箱子生命周期：可用 → 借出（绑定中） → 已归还（检查中） → 可用

## 流程

1. 宠物主在小程序预约箱子，选择取箱分舵和预约时间
2. 宠物主到分舵，分舵操作员扫箱子二维码 + 扫宠物主小程序码，确认绑定生效
3. 行程期间，箱子与该宠物主绑定，实时监测数据归属该行程
4. 行程结束，宠物主到分舵还箱，分舵扫箱子码，确认还箱，绑定解除，箱子进入"检查中"状态
5. 分舵检查箱子外观和设备状态，正常则标记为"可用"

## 理由

- **一箱不能二租**：箱子物理上同一时间只能被一个宠物主使用，必须有明确的占用和释放机制
- **数据归属**：行程期间所有传感器数据、视频片段需要归属到具体的行程记录，以便生成旅程报告
- **归还触发**：按天计租、无固定到期时间，绑定必须在分舵实际还箱时才解除，而非依赖时间推算

## 不采用方案的原因

- **预约时绑定**：如果宠物主预约后不来取，箱子空耗，且平台无法提前释放该箱子给其他用户
- **固定到期时间**：行程实际时长不确定，固定到期时间会导致提前结算或超期继续计费的复杂性
